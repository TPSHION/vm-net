# 桌面宠物功能实现文档

## 1. 目标

为 `vm-net` 增加一个可选的 `桌面宠物` 功能，使其作为悬浮胶囊的伴生展示元素，提升桌面存在感与产品辨识度。

本功能的核心目标是：

- 在不破坏现有悬浮胶囊稳定性的前提下，增加一个高质感桌面宠物
- 宠物默认显示在悬浮胶囊旁边
- 宠物与悬浮胶囊形成固定关系，可一起显示、隐藏和移动
- 宠物具备自然、生动、偏逼真的动画表现，但不干扰原有网速展示

## 2. 产品定义

### 2.1 功能形态

桌面宠物不是替代悬浮胶囊，而是作为悬浮胶囊旁边的伴生组件存在。

推荐形态：

- 一个小型高质感插画风角色
- 独立于悬浮胶囊的透明浮层
- 默认锚定在悬浮胶囊左侧或右侧

### 2.2 默认行为

- 默认关闭
- 只有在启用悬浮胶囊时，桌面宠物才允许显示
- 悬浮胶囊显示时，宠物跟随显示
- 悬浮胶囊隐藏时，宠物同步隐藏
- 拖动悬浮胶囊时，宠物一起移动

### 2.3 第一版定位

第一版建议是“高质感陪伴型桌面角色”，而不是复杂交互宠物。

优先保证：

- 显示稳定
- 跟随稳定
- 动画自然
- 不影响桌面操作

## 3. 设计原则

### 3.1 依附悬浮胶囊

桌面宠物应从属于悬浮胶囊，不建议第一版做成完全自由移动的独立宠物。

### 3.2 展示与监控解耦

宠物不直接参与网速采样，只读取已有状态数据。

### 3.3 动画质感优先

第一版优先追求“更像活的角色”的感觉，而不是低成本像素帧动画。

### 3.4 不抢焦点

宠物不应抢占主窗口焦点，不影响用户正常点击桌面和操作其他应用。

### 3.5 稳定优先于趣味

先保证多桌面、多显示器、显示/隐藏和拖动场景稳定，再考虑复杂玩法。

## 4. 范围

### 4.1 本次实现包含

- 新增桌面宠物展示能力
- 宠物与悬浮胶囊绑定显示
- 宠物跟随悬浮胶囊移动
- 宠物默认停靠在胶囊一侧
- 基于 Rive 的状态机动画
- 新增宠物开关与基础配置

### 4.2 本次实现不包含

- 宠物自由漫游桌面
- 宠物复杂对话系统
- 宠物语音
- 宠物自由拖拽独立定位
- 宠物复杂情绪树
- 多宠物同时存在

## 5. 形态方案

### 5.1 推荐方案

推荐使用：

- `独立透明浮层 + Rive 角色视图`

宠物作为一个单独的 `NSPanel` 显示，与悬浮胶囊保持固定相对偏移。

### 5.2 不推荐方案

不建议第一版采用以下方案：

- 将宠物直接绘制到悬浮胶囊内部
- 让宠物脱离胶囊独立在桌面上乱跑
- 使用 GIF / 视频 / 简单逐帧像素动画替代角色系统

原因：

- 会破坏当前胶囊的紧凑尺寸
- 会让交互和布局复杂度显著增加
- 很难做出“逼真活的感觉”

## 6. 技术选型

### 6.1 面板容器

推荐继续使用 `NSPanel`，保持和现有悬浮胶囊同一技术路线。

建议配置：

- `borderless`
- `nonactivatingPanel`
- `level = .floating`
- `isFloatingPanel = true`
- `backgroundColor = .clear`
- `isOpaque = false`
- `collectionBehavior = [.canJoinAllSpaces, .fullScreenAuxiliary]`

### 6.2 动画引擎

推荐使用 `Rive` 作为桌面宠物的动画运行时。

原因：

- 更适合高质感角色动画
- 比传统逐帧动画更自然
- 支持状态机
- 支持通过代码输入驱动动作切换
- 更适合实现“活”的感觉

推荐接入方式：

- 使用 `Swift Package Manager` 引入 `rive-ios`
- 宠物内容区通过 `AppKit` 宿主视图承载 Rive 视图

### 6.3 内容资源

Rive 资源建议形式：

- 单个 `.riv` 文件
- 一个主 `Artboard`
- 一个默认 `State Machine`

第一版不建议继续使用：

- GIF
- 视频资源
- 传统 Sprite 帧动画

## 7. 架构方案

### 7.1 推荐结构

```text
ThroughputStore
    ->
FloatingBallController
    ->
DesktopPetController
    ->
DesktopPetContentView
    ->
Rive Runtime View
```

### 7.2 类型建议

建议新增以下文件：

- `DesktopPet/DesktopPetController.swift`
- `DesktopPet/DesktopPetContentView.swift`
- `DesktopPet/DesktopPetState.swift`
- `DesktopPet/DesktopPetRiveBridge.swift`
- `DesktopPet/DesktopPetInputMapper.swift`

### 7.3 角色说明

#### DesktopPetController

职责：

- 创建宠物面板
- 控制宠物显示、隐藏
- 监听悬浮胶囊位置变化
- 更新宠物窗口位置

#### DesktopPetContentView

职责：

- 承载 Rive 动画视图
- 负责宠物自身视觉表现

#### DesktopPetRiveBridge

职责：

- 管理 `.riv` 文件加载
- 管理 `artboard`
- 管理 `state machine`
- 向 Rive 注入状态输入

#### DesktopPetInputMapper

职责：

- 将当前吞吐状态映射为宠物动作输入
- 控制 `idle / blink / busy / excited` 的切换策略

#### DesktopPetState

职责：

- 描述宠物当前状态
- 例如：
  - `idle`
  - `blink`
  - `busy`
  - `excited`

## 8. 与悬浮胶囊的关系

### 8.1 锚定逻辑

宠物不单独保存绝对位置，而是保存相对于悬浮胶囊的偏移。

建议字段：

- `petAnchorSide`
- `petOffsetX`
- `petOffsetY`

示例：

- 左侧：`(-56, -2)`
- 右侧：`(+8, -2)`

### 8.2 拖动逻辑

第一版建议：

- 只允许拖动悬浮胶囊
- 宠物自动跟随胶囊位置变化

不建议第一版允许宠物单独拖动。

### 8.3 显隐逻辑

宠物显隐应依赖于：

- `showInFloatingBall == true`
- `showDesktopPet == true`

满足这两个条件时才显示。

## 9. 动画方案

### 9.1 第一版推荐动画

第一版建议基于 Rive 状态机实现这些动作：

- 呼吸
- 眨眼
- 轻微摆头
- 轻微身体起伏

### 9.2 网速联动

可以让宠物轻微感知当前网速状态，但不建议做得太强。

建议规则：

- 低速或空闲：`idle`
- 持续下载较高：`busy`
- 上下行同时较高：`excited`

建议不要按每一秒原始数值瞬时切换，而是做轻量阈值和延迟回退。

### 9.3 状态机建议

建议第一版在 Rive 中定义一个默认状态机，包含以下输入：

- `isIdle: Bool`
- `isBusy: Bool`
- `isExcited: Bool`
- `blinkNow: Trigger`

这样运行时只需要更新少量输入，不需要频繁直接干预具体动画片段。

## 10. 视觉建议

### 10.1 尺寸

建议第一版尺寸：

- `56 x 56`
或
- `64 x 64`

尺寸不宜过大，避免喧宾夺主。

### 10.2 风格

推荐风格：

- 轻拟物小机器人
- 柔和插画小精灵
- 2.5D 吉祥物
- 小型高质感贴纸角色

不建议第一版直接使用：

- 复杂 Live2D
- 高精大体积透明视频资源

### 10.3 视觉关系

宠物和胶囊应形成“搭档感”，但不能在视觉上遮挡胶囊数值。

## 11. 配置项定义

建议在 `AppPreferences` 中新增：

- `showDesktopPet: Bool = false`
- `desktopPetSide: DesktopPetSide = .right`
- `desktopPetScale: Double = 1.0`
- `desktopPetFollowThroughputState: Bool = true`
- `desktopPetAssetName: String = "default"`

说明：

- `showDesktopPet`：是否启用宠物
- `desktopPetSide`：左侧或右侧
- `desktopPetScale`：宠物缩放比例
- `desktopPetFollowThroughputState`：是否根据网速状态切换动作
- `desktopPetAssetName`：当前使用的 Rive 角色资源

## 12. UI 方案

### 12.1 设置页

建议在设置页新增一个 `桌面宠物` 分组：

- `启用桌面宠物`
- `停靠位置`
- `宠物大小`
- `跟随网速状态变化`

### 12.2 入口关系

建议：

- 只有在悬浮胶囊已开启时，宠物开关才可用

如果胶囊未开启，可显示提示：

- `需要先开启悬浮胶囊`

## 13. 交互建议

### 13.1 第一版交互

建议仅支持：

- 右键菜单
- 悬浮胶囊拖动时跟随
- 可选轻量 hover 反应

### 13.2 不建议的交互

第一版不建议加入：

- 左键打开主窗口
- 拖动宠物脱离胶囊
- 双击切换状态
- 宠物气泡对话

## 14. Rive 资源规范

### 14.1 资源组织建议

建议在项目中增加：

```text
vm-net/Resources/DesktopPet/
  default/
    pet.riv
    manifest.json
```

### 14.2 `.riv` 规范建议

建议统一约束：

- 一个主 `Artboard`
- 一个默认 `State Machine`
- 命名稳定
- 角色默认朝向固定
- 透明背景

### 14.3 首版资源要求

第一版资源建议满足：

- 小尺寸下轮廓清晰
- 深色桌面背景下仍有识别度
- 在 `56x56` 或 `64x64` 尺寸下仍可读
- 呼吸和眨眼动作自然

## 15. 多屏与空间处理

### 15.1 多显示器

宠物应始终跟随胶囊所在屏幕，不单独做屏幕定位。

### 15.2 多桌面空间

建议与胶囊保持一致：

- `canJoinAllSpaces`
- `fullScreenAuxiliary`

### 15.3 边界修正

如果宠物停靠后超出屏幕边缘，应自动切换到另一侧，或调整偏移量。

## 16. 性能与风险

### 16.1 风险点

需要特别注意：

- Rive runtime 引入后的包体和依赖管理
- 透明浮层加动画的长期运行稳定性
- 状态机输入频繁更新导致不必要重绘
- 多屏切换时位置关系错乱

### 16.2 控制原则

- 只有启用宠物时才初始化 Rive runtime 视图
- 只保留一个宠物实例
- 仅在状态变化时更新输入
- 避免同时叠加复杂阴影和过大尺寸

## 17. 分阶段实施建议

### 17.1 Phase 1

- 新增 `DesktopPetController`
- 透明宠物面板
- 与悬浮胶囊绑定显示/隐藏
- 跟随胶囊位置变化
- 引入 Rive runtime
- 跑通默认 `.riv` 资源展示
- 支持 `idle` 状态

### 17.2 Phase 2

- 接入状态机输入
- 支持左侧/右侧停靠切换
- 支持大小配置
- 支持随网速状态切换动作

### 17.3 Phase 3

- 加入轻量 hover 反应
- 自动边缘翻转
- 补充更多宠物主题资源

## 18. 验收标准

完成功能后应满足：

- 宠物可在设置页开启/关闭
- 宠物显示在悬浮胶囊旁边
- 胶囊移动时宠物同步移动
- 胶囊隐藏时宠物同步隐藏
- 多桌面和多显示器下位置关系稳定
- 长时间运行不出现明显 CPU 异常
- 不影响悬浮胶囊速率展示

## 19. 推荐落地结论

当前项目最适合的第一版方案是：

- 桌面宠物作为 `悬浮胶囊旁边的独立透明面板`
- 使用 `Rive` 作为动画引擎
- 使用单个 `.riv` 角色资源
- 默认关闭
- 与悬浮胶囊绑定显示、隐藏和移动
- 通过状态机驱动 `idle / busy / excited`
- 先不做自由漫游和复杂交互

这样最符合 `vm-net` 当前的产品形态，也最容易在现有悬浮胶囊架构上稳定落地。
